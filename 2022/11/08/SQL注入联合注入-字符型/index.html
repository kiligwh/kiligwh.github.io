SQL注入-联合注入【字符型】

概述

注入的本质

用户输入的数据被当做代码执行

注入条件

用户可以控制传参

用户输入的语句被带入数据库并执行

显错注入原理

用户输入的数据被当做SQL语句执行，并带入数据库执行

对用户输入的数据没有做任何过滤，并且将结果返回到页面

没有做任何防护

注入的关键操作

构造闭合

注入基础

MySQL注释符：# 或 –空格 或 /***/
内联注释：/**! SQL语句 */ 只有MySQL可以识别，常用来绕过WAF
注入探测符：常使用 \ 进行探测
